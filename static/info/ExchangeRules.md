## Отправка значений переменных (конфиг устройства, изменения параметров)

Чтобы отобразить значения параметров устройства в Web-интерфейсе необходимо отправить пакет следующего вида:

```
{
  HEADER: OK!,
  ARGUMENT: Data,
  VALUE: {
    DevSN: '00.00.000-00_0000AC670000B22B00007D88:43',
    ModSN: '00.00.000-00_0000AC670000B22B00007D88:43',
    Title: 'System',
    Message: 'Device Connected to Cloud',
    Payload: {...}
  }
}

```

- Поле `HEADER` должно принимать значение _OK!_ или _ER!_
- Поле `ARGUMENT` должно принимать значение _Data_ или _DataStream_ (_DataStream_ предпочтительно использовать при большом потоке подобных данных, типа точек графиков или прогресса, так как пакеты под этим заголовком не сохраняютя в базу данных)
- Поле `Title` - заголовок статуса
- Поле `Message` - описание статуса
- Поле `Payload` - объект с перечнем всех переменных, которым нужно обновить значение в интерфейсе, либо _null_

### Для отправки _ПРОГРЕССА_ нужно использовать переменную, например `Progress`, которая содержит массив чисел:

```
Payload: {
  Progress: [10, 0, 99]
}
```

- Массив содержит столько значений, сколько полос прогресса содержит компонент. Допускается передача числа, если массив состоит из 1 элемента

### Для отправки _ЦВЕТА_ нужно использовать переменную, например `Color`, которая содержит массив из 3 чисел, соответствующих цветовой модели RGB:

```
Payload: {
  Color: [10, 0, 99]
}
```

### Для отправки _ТОЧКИ ГРАФИКА_ нужно использовать переменную, например `Graph`, которая содержит массив объектов следующего формата:

```
Payload: {
  Graph: [
    { name: 'roll', value: 51, timestamp: (Date) },
    { name: 'pitch', value: 92, timestamp: (Date) }
  ]
}
```

- Массив содержит столько значений, сколько графиков нужно отобразить в компоненте
- Поле `name` содержит имя графика, к которому пришло значение
- Поле `value` содержит значение параметра
- Поле `timestamp` содержит временную метку. Данная метка нужна для того, чтобы компонент графика работал в автоматическом режиме - отображал данные по их появлению, а не считывал их с заданной в нем частотой
